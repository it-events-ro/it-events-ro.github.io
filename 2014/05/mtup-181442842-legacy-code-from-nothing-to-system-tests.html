Legacy code: From nothing to System Tests

vineri, 16 mai 2014

What


This session will be a performing code kata followed by a discussion. Join to learn one legacy code technique and ask difficult questions to the group. We can write code after the session if you want, so bring your laptop.


Why

Whenever I need to change an existing system I need a safety net. These system tests knit a coarse safety net, very good if you want to have the safety of changing code later without introducing defects.

How

I will write some code on the projector for 45 minutes to one hour. The rest of the session we can discuss legacy code and techniques that we can apply.

The first thing I usually do when I start working with a system that:

• does not have any automated tests

• is totally unknown to me

is to write some automated tests is such a way that the System Under Test (SUT) will not be changed at all, not even touched.


This safety net will be used during the next phases when the code will be refactored and cleaned-up, before being modified.

Please come prepared with difficult questions!
