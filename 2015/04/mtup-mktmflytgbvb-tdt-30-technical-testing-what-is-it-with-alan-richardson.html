TdT #30 - Technical Testing - What is it? - with Alan Richardson

miercuri, 1 aprilie 2015

Hello everyone,

This will be another extraordinary meeting, as we will have a special guest: Alan Richardson, @eviltester.

*Content

The subject of the webinar will be Technical Testing - What is it?

Alan uses the term Technical Testing to describe what he does when he tests. But what does Technical Testing mean? For Alan, it means going beyond requirements and using Technical Information about the implementation and an understanding of the technologies used in the building of the system to add to the risk profile and use to help derive test approaches. In this webinar Alan will explain what he understands by Technical Testing, using Web Testing as an example. He will explain how approaching testing from a technical perspective changes how you view the system and how you test. He will also explain how a technical understanding leads to a different use of tooling an automation.

Alan will answer questions during the webinar, but if you have any questions in advance please add them in the comments below and Alan can weave them into the webinar early.
